## KI-Nutzungsbericht

### Vollständige Prompts

Design:
"is my design for the app fire?"

Programmierung:
"Explain routers in react native"
"is there a way i could stylize the switch element in react native beyond that what can be set with props?"
"is my filter structure reasonable? [code]"

Debugging:
"why does this not work?"
"im trying to X, and i get this" und ähnliche

### Alle relevanten Prompt-Anpassungen/Iterationen

"do you think my design is usable?"
"and to remove the green thumb when its activated?"
"are the layouts visually pleasing?"
"generate user flows to test the usability"

### Kennzeichnung der übernommenen Code-Teile

Wurden im Code mit Kommentar "Völlig von KI übernommen" oder "Teils von KI übernommen" markiert.

### Kurze Reflexion

KI wurde vor allem dafür benutzt um das Lernen von React Native Konzepten zu erleichtern und Debugging zeiteffizienter durchzuführen. Außerdem habe ich es auch bei der Design-Phase eingesetzt um ein Feedback von dem Interfacedesign zu bekommen.

Einige Teile von dem Code wurden von KI gelöscht, wie zum Beispiel Komponentenfunktionen, weil die zwar vielleicht Sinn aus der Designersicht, aber nicht aus der React Native Sicht gemacht haben. Einige Funktionen wie zum Beispiel API-Fetch-Funktion wurde leicht angepasst.

Beim Benutzen von KI merkte ich grundsätzlich keine Fehler oder Limitationen. Jedoch es war schwer mein genaues Projektverzeichnis mit der ganzen Struktur, inklusive einzelnen Komponentenseiten, dem bekanntzugeben.
